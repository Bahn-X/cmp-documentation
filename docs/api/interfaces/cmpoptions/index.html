cmpoptions

@bahn-x/cmp-sdk / CmpOptions

Interface: CmpOptions#

DB CMP SDK options.

Options to configure the consent layer UI behaviour. Have to be placed in an existing utag_data object.

Note: To avoid naming conflicts with other utag_data properties, all options are prefixed with cmp_.

example

const options: CmpOptions = { cmp_disable_reload: true }
window.utag_data = { ...options };

Disables the page reload after a consent change event happened. Note: If you disable reload, you have to make sure queued events that affect user data are stopped if consent has been revoked.

default undefined (Page reloads on consent changes, except for the very first user interaction.)
